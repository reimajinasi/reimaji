# 0.8 Create Development Workflow – Implementasi

## 0.8.1 Git Branching Strategy (GitHub Flow)
- Branch utama: `main` (stabil, siap rilis)
- Feature branch: `feature/<slug>` untuk fitur/bugfix
- Proses: buka PR → code review → squash merge ke `main`
- Aturan: satu PR kecil, lint/type-check hijau, deskripsi jelas, referensi task

## 0.8.2 Development Scripts
- `npm run dev`: Next.js frontend
- `npm run dev:backend`: Convex backend
- `npm run dev:setup`: jalankan keduanya paralel (Next + Convex)
- `npm run lint`, `npm run lint:fix`
- `npm run type-check`
- `npm run format`, `npm run format:check`

## 0.8.3 Code Review Checklist
- Lint & type-check hijau; build tidak error
- Aksesibilitas: label, role, fokus, warna kontras
- Konsistensi design system: tokens, komponen `ui/*`, layout `Container/Section`
- Keamanan: jangan commit secrets; input tervalidasi; tidak ada log sensitive
- Readability: nama jelas, fungsi kecil, tidak ada komentar rahasia
- Performa: hindari kerja berat di client; gunakan server components saat bisa
- Testing: tambahkan test bila ada logic penting

## Aksi yang dilakukan
- Tambah script: `dev:backend`, `dev:setup`, `lint:fix`, `format:check` di `package.json`
- Verifikasi perintah: type-check, lint, build

## Bukti
- `npm run type-check` → OK
- `npm run lint` → OK
- `npm run build` → OK
