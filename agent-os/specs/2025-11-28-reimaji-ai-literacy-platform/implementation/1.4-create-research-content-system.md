# 1.4 Create Research Content System

## Tujuan
- Listing Research dengan filter tag dan pencarian.
- Halaman detail Research menampilkan implikasi + gating Premium.

## Implementasi
- Convex queries: `research.list`, `research.getBySlug`, `research.listAll`.
- Convex mutations: `research.create`, `research.update`, `research.remove` (RBAC admin/superadmin).
- UI listing: `src/app/(public)/research/page.tsx` (server component, filter via searchParams).
- UI detail: `src/app/(public)/research/[slug]/page.tsx` (gating premium, implikasi + konten).

## Verifikasi
- `npm run type-check` → OK
- `npm run lint` → OK
- `npm run build` → OK; route `/research` dan `/research/[slug]` terdeteksi.

## Bukti
- Build output mencantumkan route Research.
- Gating sesuai role user.

## Risiko
- Kategori menggunakan `tags`; kategori khusus bisa ditambah kemudian di schema.
