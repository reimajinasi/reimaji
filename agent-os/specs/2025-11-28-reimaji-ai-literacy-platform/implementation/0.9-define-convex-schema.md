# 0.9 Define Convex Schema – Implementasi

## Aksi
- Tambah tabel `courses`, `modules`, `lessons` dengan soft delete (`deletedAt`) dan indeks kinerja.
- Pertahankan `users`, `news`, `research` dengan indeks publish/isPremium.
- Jalankan update Convex satu kali untuk generate types dan push skema.

## Rincian Skema
- `courses`: `title`, `slug`, `description`, `imageUrl`, `isPublished`, `publishedAt`, `createdBy`, `tags`, `deletedAt`, `createdAt`, `updatedAt` + indeks (`by_slug`, `by_isPublished`, `by_createdBy`).
- `modules`: `courseId`, `title`, `order`, `description?`, `deletedAt`, `createdAt`, `updatedAt` + indeks (`by_course`, `by_order`).
- `lessons`: `moduleId`, `title`, `order`, `content`, `videoUrl?`, `isPublished`, `publishedAt?`, `deletedAt`, `createdAt`, `updatedAt` + indeks (`by_module`, `by_order`, `by_isPublished`).

## Bukti
- Convex push satu kali: `npx convex dev --once --typecheck disable --codegen enable` → sukses, indeks tercatat.
- `npm run type-check` OK, `npm run lint` OK, `npm run build` OK.
