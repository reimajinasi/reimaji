# 1.1 Complete User Authentication Flow

## Tujuan
- Registrasi dengan verifikasi email via Clerk.
- Dashboard user dengan info profil dasar.
- Fungsi reset password tersedia via flow Clerk (Forgot password di SignIn).

## Langkah Implementasi
- Proxy global: `src/proxy.ts` pakai `clerkMiddleware` dan `config.matcher` untuk proteksi.
- Halaman SignIn/SignUp: sudah ada (`/sign-in`, `/sign-up`) menggunakan komponen Clerk.
- Halaman Dashboard: `src/app/(app)/dashboard/page.tsx` disinkronkan ke Convex (`users.upsertFromClerk`), tampilkan `email` dan `role` dari DB.
- Halaman Profil: `src/app/(app)/profile/page.tsx` menggunakan `<UserProfile />` untuk manajemen profil.
- Reset password: gunakan tautan "Forgot password" di `SignIn` Clerk (menyediakan alur reset via email).

## Verifikasi
- `npm run type-check` → OK
- `npm run lint` → OK
- `npm run build` → OK, routes: `/dashboard`, `/profile`, `/sign-in`, `/sign-up`, `/sign-out` terdeteksi.

## Bukti
- Build sukses; daftar route termasuk `/dashboard` dan `/profile`.
- Dashboard menampilkan `Email` dan `Role` dari Convex setelah sinkronisasi.

## Risiko & Mitigasi
- Kunci Clerk belum diisi → pastikan `.env.local` berisi publishable/secret keys.
- Penggunaan `middleware.ts` usang → sudah dihapus dan diganti `src/proxy.ts` sesuai Next.js 16.
