# 1.3 Create News Content System

## Tujuan
- Listing News dengan filter (q, type, tag).
- Halaman detail News dengan gating (Free vs Premium).

## Implementasi
- Convex queries: `news.list`, `news.getBySlug`, `news.listAll`.
- Convex mutations: `news.create`, `news.update`, `news.remove` (RBAC admin/superadmin).
- UI listing: `src/app/(public)/news/page.tsx` (server component, searchParams → filter, kartu konten).
- UI detail: `src/app/(public)/news/[slug]/page.tsx` (auth + role dari Convex, konten terkunci jika premium).

## Verifikasi
- `npm run type-check` → OK
- `npm run lint` → OK
- `npm run build` → OK; route `/news` dan `/news/[slug]` terdeteksi.

## Bukti
- Build output mencantumkan route News.
- Gating menampilkan ringkasan jika user `guest/free` untuk konten premium.

## Risiko
- Belum ada data awal → tambahkan via Admin panel.
- Sanitasi HTML konten ditangani oleh author; bisa ditingkatkan nanti.

- [x] Detail + gating
