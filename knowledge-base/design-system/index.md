# Reimaji – Design System (Index)

Design system Reimaji dibagi menjadi beberapa dokumen kecil agar mudah dibaca dan diimplementasikan oleh agent maupun developer:

- `design-typography.md` – Typography (font family, heading, body, caption/label, mapping ke Tailwind).  
- `design-colors.md` – Palet warna (tokens, penggunaan, mapping ke Tailwind/shadcn).  
- `design-shadows.md` – Shadows (elevation scale 100–800, penggunaan, mapping ke Tailwind).  
- `design-button.md` – Aturan desain Button (size, variant, state, mapping ke komponen shadcn).  
- `design-button-group.md` – Aturan desain Button Group (grouping, segmented control, layout, state).  
- `design-input.md` – Aturan desain Input (text field, textarea, states, label/helper/error, icons).  
- `design-badge-chip.md` – Aturan desain Badge & Chip (status label, filter chips, variants).  
- `design-icons.md` – Aturan desain Icons (ukuran, warna, penggunaan di Button/Input/Badge/Chip).  
- `design-alert.md` – Aturan desain Alert (status info/success/warning/error, layout, mapping ke komponen Alert).  
- `design-navbar-bottom.md` – Aturan desain Bottom Navbar (navigasi utama mobile, item aktif/inaktif, layout).  
- `design-navbar-top.md` – Aturan desain Top Navbar (header utama, nav links, actions, responsif).  
- `design-checkbox.md` – Aturan desain Checkbox (state, warna, layout, mapping ke komponen).  
- `design-radio.md` – Aturan desain Radio (state, warna, layout, mapping ke komponen).  
- `design-toggle.md` – Aturan desain Toggle/Switch (on/off setting, state, mapping ke komponen).  
- `design-avatar.md` – Aturan desain Avatar (foto profil, inisial, status badge, mapping ke komponen).  
- `design-progress-bar.md` – Aturan desain Progress Bar (progress linear, ukuran, warna, mapping ke komponen).  
- `design-breadcrumbs.md` – Aturan desain Breadcrumbs (hierarki navigasi, layout, state).  
- `design-tooltips.md` – Aturan desain Tooltips (informasi tambahan singkat, warna, posisi).  
- `design-loader.md` – Aturan desain Loader/Spinner (loading indeterminate, ukuran, warna, penggunaan di komponen).  
- `design-popup.md` – Aturan desain Pop-Up/Modal (overlay, dialog, actions, mapping ke komponen Dialog).  
- `design-list.md` – Aturan desain List (struktur item, density, hover/selected).  
- `design-action-sheet.md` – Aturan desain Action Sheet (panel aksi bawah, mobile).  
- `design-context-menu.md` – Aturan desain Context Menu (menu konteks klik kanan/long-press).  
- `design-dropdown.md` – Aturan desain Dropdown (select-like, panel opsi, states).  
- `design-card.md` – Aturan desain Card (kontainer konten, variants).  
- `design-tabs.md` – Aturan desain Tabs (navigasi konten dalam satu halaman).  
- `design-pagination.md` – Aturan desain Pagination (navigasi halaman data, layout & states).  
- `design-page-control.md` – Aturan desain Page Control (indikator slide/dot).  
- `design-stepper.md` – Aturan desain Stepper (langkah multi-step, status step).  
- (ke depan) Dokumen per komponen lain: `design-forms.md`, dsb.

Prinsip umum:
- Semua keputusan UI harus konsisten dengan Figma Design System (`Design-System--Community-`).  
- Di kode, selalu gunakan token dari dokumen ini (typography, colors, shadows) – jangan membuat style bebas.  
- Komponen dan layout harus reusable, mengikuti pola di `frontend-system-design.md`.
